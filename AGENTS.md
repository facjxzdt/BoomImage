# AGENTS.md

本文件适用于整个 BoomImage 仓库。所有参与实现、审查或测试的 Agent 在开始工作前都应阅读本文件和 [PROJECT_PLAN.md](./PROJECT_PLAN.md)。

## 项目定位

BoomImage 是一个面向单用户的高性能图床，目标是：

- 单机 Docker Compose 一键部署。
- 保留原图，并异步生成 AVIF、WebP 和缩略图。
- 本地媒体读取由 Caddy 直接提供；S3 媒体可选择对象存储/CDN 直链或 BoomImage 服务器代理。
- 默认使用本地文件系统和 SQLite，保持部署及备份简单；可按上传选择 S3 兼容对象存储。

详细架构、数据模型、API 草案和里程碑以 `PROJECT_PLAN.md` 为准。

## 默认技术栈

- Node.js LTS、TypeScript、pnpm workspace。
- Fastify API。
- Sharp/libvips 图片处理。
- SQLite WAL 模式。
- React + Vite + Tailwind CSS 管理界面。
- Caddy 静态媒体服务和 HTTPS。
- 可选 S3 兼容对象存储。
- Docker Compose 部署。

除非需求明确改变，不要引入 Redis、PostgreSQL、微服务、Kubernetes 或任意参数的动态图片转换接口。S3 兼容存储只作为外部存储后端接入，不在默认 Compose 中编排 MinIO 服务。

## 实现原则

- 优先完成可验证的端到端功能，不提前搭建无实际用途的抽象层。
- 上传必须流式落盘，不能按上传文件大小在线性占用应用内存。
- 图片文件先写入 `/data/tmp` 并完成校验；本地存储发布时通过同文件系统的原子重命名完成，S3 存储发布时由服务端上传到服务端生成的对象 key。
- 原图必须保留；转换失败不能破坏或删除原图。
- 图片二进制存储在本地文件系统或 S3 兼容对象存储，SQLite 只保存元数据、路径、存储后端和任务状态。
- 转换任务必须持久化、幂等、可重试，并能在进程重启后恢复。
- 转换并发必须受控，不能为每个上传无限创建线程或任务。
- 公开媒体使用内容哈希路径和不可变缓存。
- 本地普通媒体读取应由 Caddy 完成，不经过 Fastify 或 Sharp；S3 直链由对象存储/CDN 提供，S3 proxy 只允许代理数据库中存在且未删除的媒体路径。
- 文件路径必须由服务端生成，不能直接拼接用户提供的文件名。

## 安全约束

- 不信任扩展名或客户端 MIME；根据文件内容识别并实际解码验证。
- 限制上传大小、解码像素、multipart 字段数、文件数和处理超时。
- 默认移除 EXIF、GPS 等隐私元数据，并正确处理 EXIF 方向。
- SVG 第一版只保存，不自动栅格化；动图第一版只保留原图。
- 管理密码使用 Argon2id；API Token 只保存哈希且明文只展示一次。
- Cookie 鉴权使用 `HttpOnly`、`Secure`、`SameSite=Strict`，写操作提供 CSRF 防护。
- 日志和错误响应不得包含密码、Token、S3 密钥、宿主机路径或内部堆栈。
- 删除或清理文件前必须验证最终路径位于预期数据目录内。
- S3 对象 key 必须由服务端生成并按前缀约束，不能代理任意用户提供的 key。

## 代码约定

- TypeScript 开启严格模式，避免无说明的 `any`。
- 在 API 边界执行运行时校验，不能只依赖 TypeScript 类型。
- 数据库结构通过迁移管理，不在应用启动时零散执行临时 DDL。
- 数据库时间统一使用 UTC ISO 8601；资源 ID 使用 ULID。
- API 错误采用稳定的错误码和结构，不将内部异常直接返回客户端。
- 复杂业务逻辑与 HTTP handler 分离；handler 负责鉴权、校验和响应映射。
- 对外行为变化应同步更新 API 文档、环境变量示例和相关测试。
- 新增依赖前确认用途、许可证、维护状态及 Docker 平台兼容性。

## 测试与验证

每项变更应执行与风险相称的验证，并在交付时列出实际运行的命令。

最低要求：

- 代码格式检查和 TypeScript 类型检查。
- 受影响模块的单元测试。
- API 变更需包含 Fastify 集成测试。
- 数据库变更需验证全新迁移和已有数据库升级。
- 图片处理变更至少覆盖 JPEG、PNG、WebP、AVIF、透明图片、EXIF 方向、超大图片和损坏文件。
- 任务队列变更需验证重试、租约过期、幂等和进程重启恢复。
- Docker 或 Caddy 变更需构建镜像并验证健康检查、上传和静态读取闭环。
- S3 存储变更需验证 local 兼容、S3 direct URL、S3 proxy 公开读取、worker 从 S3 读取原图并写回变体、删除和未配置 S3 的错误路径。

不要为了让测试通过而降低安全限制、删除失败用例或吞掉异常。

一项任务只有在以下条件满足后才视为完成：

1. 功能符合请求和 `PROJECT_PLAN.md` 的相关约束。
2. 错误路径和安全边界得到处理。
3. 相关测试、类型检查或部署验证已经实际运行。
4. 文档和配置示例与实现保持一致。
5. 没有遗留临时文件、硬编码密钥或调试输出。
